# Archirectural decision record

Date: 2023-10-19

## Status

Accepted

## Context

We need to record the architectural decisions made on this project.

There are several ways to do that, but a simple way is adopt ADR.

It's allow async work, allowing everyone to propose changes and
get feedback before implement complex features.

## Decision

We will use Architecture Decision Records, as described by Michael Nygard in this article: http://thinkrelevance.com/blog/2011/11/15/documenting-architecture-decisions

## Consequences

See Michael Nygard's article, linked above.
